# CHANGELOG V2.3.3

## 数据结构

- 新增即时错词池：`currentWrongWords`。
  - 用于“本轮纠错”。
  - 只保存本轮听写或纠错中尚未解决的词。
  - 生成新学习词库时清空。

- 长期错词库：`wrongWordBank`。
  - 用于“薄弱词库”。
  - 长期保存历史薄弱词。

```js
wrongWordBank: {
  tractor: {
    errorCount: 5,
    lastWrong: "2026-07-12T00:00:00.000Z",
    correctStreak: 2,
    mastered: false
  }
}
```

- 兼容旧数据：旧的 `wrongWords` 会迁移读取到 `wrongWordBank`。

## 名称调整

- “错词复练”改名为“本轮纠错”。
- “错词复习”改名为“薄弱词库”。
- “错误次数”类文案改为更适合家长理解的表达：
  - 曾错误：8 次
  - 连续正确：2 次
  - 状态：待加强 / 已掌握

## UI调整

- 顶部主操作区只保留：
  - 生成学习
  - 开始听写
  - 再读一次
  - 下一词
- “本轮纠错”入口移动到批改结果区域。
- “重新听写”入口移动到批改结果区域。
- “查看薄弱词库”入口移动到学习档案区域。
- 批改结果增加摘要：正确 x / 总数，错误 y。

## 用户流程优化

- 完成批改后，如果有本轮错误，结果区显示“本轮纠错”。
- 本轮错误解决后，显示“本轮没有需要纠正的单词”。
- 学习档案显示“薄弱词”数量，并提供“查看薄弱词库”。
- 没有长期待加强词时，提示“当前没有需要重点复习的词”。
